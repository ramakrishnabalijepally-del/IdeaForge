02 / Prompt Engineering
Master Prompt & AI Prompt Design
Two carefully engineered system prompts power the AI features — one for idea generation (creative, temp 0.7) and one for RAG search (factual, temp 0.3).
AI Idea Generator — System Prompt (backend/app/services/ai_generator.py)
SYSTEM: You are an expert startup and manufacturing business consultant.
Generate a detailed, realistic business idea based on the user's niche or keyword.
Return ONLY valid JSON matching this exact schema:

{
  "title": "string",
  "problem_statement": "string",
  "proposed_solution": "string",
  "target_market": "string",
  "revenue_model": "string",
  "competitive_landscape": "string",
  "estimated_capital": "string",
  "feasibility_score": 1-10,
  "technical_difficulty": "low|medium|high",
  "tags": ["string"],
  "execution_roadmap": [
    "Step 1: ...", "Step 2: ...", "Step 3: ...",
    "Step 4: ...", "Step 5: ..."
  ]
}

Rules:
- feasibility_score must be 1-10 (integer)
- technical_difficulty must be exactly "low", "medium", or "high"
- execution_roadmap must have exactly 5 actionable steps
- Be specific and realistic — no generic placeholders
- Output ONLY the JSON object, no markdown, no explanation
RAG Search — System Prompt (backend/app/services/rag_service.py)
SYSTEM: You are IdeaForge's AI search assistant. Your sole purpose is to help
users discover relevant startup and manufacturing business ideas.

Format rules:
1. One sentence explaining what relevant ideas were found for the query.
2. Each idea: **[Idea Title]** — why it's relevant + feasibility + capital.
3. A brief closing recommendation (1-2 sentences max).
4. Only surface ideas from context — do NOT invent ideas.
5. Under 350 words. Use markdown bold for idea titles.

Temperature: 0.3 (factual, grounded — not creative)
Generator temp: 0.7 (creative variation for novel ideas)
Prompt Engineering Decisions: The generator uses strict JSON schema enforcement to prevent hallucinated structures. The RAG prompt explicitly forbids inventing ideas and enforces citation of source titles. Separate temperatures (0.7 vs 0.3) balance creativity vs factual grounding for their respective tasks.

04 / AI-Assisted Development & AI Features
Which AI Was Used & Why
Built entirely with Claude Code. Three AI techniques selected based on domain fit — all others explicitly evaluated and excluded with justification.
🤖
LLM — Gemini 3.1 Flash Lite
✓ Used
Powers the AI Idea Generator. Given a niche keyword, returns a fully structured JSON business report (problem, solution, market, revenue model, feasibility 1-10, 5-step roadmap). Temperature 0.7 for creative variation.
📊
RAG — ChromaDB + Gemini Embeddings
✓ Used
Idea database is a text knowledge base — perfect for RAG. Gemini Embedding 001 creates 1536-dim vectors. ChromaDB retrieves top-5 by cosine similarity. Gemini generates grounded answers with citations.
🎯
ML Recommendation — TF-IDF Cosine
✓ Used
scikit-learn TF-IDF vectorises all ideas (title + problem + solution + tags). User's saved-idea profile is the mean vector. Cosine similarity ranks unsaved ideas. Lightweight, explainable, no training pipeline needed.
📷
Computer Vision / OCR / Speech
✗ Not Used
Domain is purely textual. No images, PDFs, or audio in the user workflow. Would add infrastructure cost without user value. Explicitly excluded.
Claude Code was used to build the entire application — all backend Python, all frontend TypeScript, all tests, configuration, and documentation. Every prompt was iteratively refined: schema enforcement was added after early Gemini responses omitted fields; auth cookie SameSite was changed from lax to none after cross-domain testing revealed cookie loss.
